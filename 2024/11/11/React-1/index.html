从JSX到DOM

JSX代码 => Babel => React.createElement => ReactElment => 虚拟DOM =>ReactDOM.render() => 真实DOM

Q1 为什么要使用JSX语法？

A1 JSX语法本质上是JS语法的拓展，经过Babel处理，会变为JS语法。这个JS语法，调用React.createElement，返回一个ReactElement。

Q2 既然如此，为什么不直接用React.creatElement来进行React元素的创建？

A2 你怎么不写机器码

Q3 React.createElement干了什么

A3 入参 => 处理 => 返回

入参：type、config、children。

type:标识节点类型（如”h1”,”div”）

config:以对象形式传入，属性以键值对的形式存储在其中(如 key = 1010 className = app)

children:子节点子元素的内容   

  

处理：二次处理key,ref,self,source四个属性值 => 遍历config，筛选可以提进props的属性 => 提取子元素，推入childArray (也即props.children)数组 => 格式化defaultProps => 结合以上数据进行入参，发起ReactElement调用

返回：ReactElement(type,key,ref,self,source,ReactCurrentOwner.current,props)

总结：将数据格式化为符合ReactElement参数的形式，传入后返回React对象

Q4 这个ReactElement又是干什么的

A4 将数据组装为一个ReactElement对象实例并返回。这个对象实例，本质上是一个描述DOM节点的JS对象，也就是虚拟DOM

Q5 虚拟DOM如何转换为真实DOM

A5 使用ReactDOM.render。

ReactDOM.render接收3个参数，Element,container,[callback]，

Element就是需要渲染的元素，container就是渲染的挂载节点，callback回调函数，可以来处理渲染结束后的逻辑

总结：

JSX => Babel => React.createElement => ReactElement => 虚拟DOM => ReactDOM.render => 真实DOM
